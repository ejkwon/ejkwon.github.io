제공 환경

  • Redis 광호스팅 소개
    • Redis 광호스팅은 Redis 구축이 어려운 호스팅 사용자를 위한 서비스입니다.
    • Redis는 Key와 Value를 쌍의 형태로 테이터를 속도가 빠른 메모리에 저장하는 메모리 DB이며, 데이터를 빠르게 저장하고 쉽게 활용하기 위한 서비스입니다.
    • 인스탄트그램, 네이버 재팬의 LINE 메신져 서비스, StackOverflow, Blizzard, digg 등 여러 소셜 서비스에 널리 사용되고 있습니다.
  • Redis 광호스팅의 장점
    • 제공된 메모리 용량 내에서 독립적으로 작동하여 서비스 구동 안정성이 보장됩니다.
    • 다양한 언어(C, C++, C#, JAVA, JavaScript, Perl, PHP, Node.js, Action Script, Clojure, Erlang, Objective-C, Python,Smalltalk,Tcl)에서 구현 가능하여 언어의 제약이 없습니다.
    • 메모리를 활용하면서 영속적인 데이터 보존이 가능합니다.
  • Redis 광호스팅의 백업
    • Redis 광호스팅은 SNAPSHOTTING + AOF 백업을 제공합니다.
    • SNAPSHOTTING (RDB) 방식
      순간적으로 메모리에 있는 내용을 DISK에 전체를 옮겨 담는 방식이며, 아래와 같이 분류됩니다.
      • SAVE 방식
        blocking 방식으로 순간적으로 redis의 모든 동작을 정지시키고, 그때의 snapshot을 disk에 저장합니다.
      • BGSAVE 방식
        non-blocking 방식으로 별도의 process를 띄운후, 명령어 수행 당시의 메모리 snaopshot을 disk에 저장하며, 저장 순간에 redis는 동작을 멈추지 않고 정상적으로 동작합니다.
    • AOF 방식
      • AOF(Append On File) 방식은 redis의 모든 write/update 연산 자체를 모두 log 파일에 기록하는 형태입니다.
      • 서버가 재 시작될때 기록된 write/update operation을 순차적으로 재 실행하여 데이터를 복구합니다.
      • operation 이 발생할때 마다 매번 기록하기 때문에, RDB 방식과는 달리 특정 시점이 아니라 항상 현재 시점까지의 로그를 기록할 수 있으며, 기본적으로 non-blocking call입니다.

서비스 이용

서비스 이용
월사용료 혜택
  • 신청하시는 서비스 기간에 따라 월사용료 할인 혜택을 제공합니다.
    6개월(5%), 12개월(10%), 24개월(12%), 36개월(15%), 120개월(20%)
서비스 변경/추가
  • 서비스를 사용하다 용량이 부족한 경우, 상위 서비스로의 변경이 가능합니다.
    (하위 서비스로의 변경은 불가능)
서비스 제한
  • 서버 부하가 과도한 계정은 서비스가 취소될 수 있습니다.
Redis 참고 링크
  • Redis 안내

서비스 주의사항

서비스 주의사항
주의사항
  • 세션정보등 임시 캐시용도로 사용하시기 바랍니다.
  • 이중으로 캐시가 되는 파일의 캐시 및 데이터베이스 쿼리 결과의 캐시로는 사용하지 마십시오.
  • 휘발성 자료만 사용하시고 중요한 자료는 이용하지 마십시오.
  • 사용량 초과 시 데이터를 삭제하시기 바랍니다.
서비스 만료
및 차단
  • 서비스 만료는 만료일 익일 오전 10시에 만료 차단을 실시합니다.
  • 차단 시 나의서비스관리 내 "접근 ip 관리" 메뉴가 비활성화됩니다.
  • 만료일 + 8일에는 데이터가 모두 삭제됩니다.
미지원 서비스
  • 하드용량, 트래픽용량, DB, POP메일계정, 서브도메인, 프로그램자동설치 등 웹호스팅 서비스에 특화된 기능들은 제공되지 않습니다.
