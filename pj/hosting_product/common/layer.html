#content와 형제레벨로 놓임

하드(HDD) 용량이란?

하드(HDD) 용량

  • 하드(HDD)는 FTP를 통해 올릴 수 있는 자료(파일)의 총 용량입니다.
    그외 게시판을 사용하는 경우 텍스트를 제외한 이미지, 동영상, 음악 등의 파일은 하드 용량에 포함됩니다.
  • 카페24호스팅은 일반적인 웹 공간만을 제공하는 타사와 달리, 스트리밍 및 CDN 서비스를 추가로 제공하여 사실적 용량증가 효과 및 고급서비스를 무료로 이용하실수 있는 기회를 제공해 드립니다.

웹 용량

  • 웹페이지 또는 동영상, 이미지 파일 등을 올릴 수 있는 용량입니다.

스트리밍 용량

  • 동영상이나 음악 파일(WMV, WMA, ASF, MP3) 등을 올릴 수 있는 무료 스트리밍 서비스 용량입니다.
  • 무료 스트리밍 서비스는 [나의서비스관리]에서 신청할 수 있습니다

CDN 용량

  • 이미지 파일(JPG,GIF,PNG)을 올릴 수 있는 무료 이미지CDN 서비스 용량입니다.
  • 무료 이미지CDN 서비스는 [나의서비스관리]에서 신청할 수 있습니다

POP E-mail 계정 안내

  • 메일 계정은 아웃룩 등의 프로그램으로 설정하여 이용하실 수 있는 서비스로 계정당 20MB의 용량이 지원됩니다.
  • 메일 계정의 개수는 옵션으로 추가하실 수 있습니다.

서비스 선택

  • 일부 오래된 cgi 등 32bit 환경이 꼭 필요하지 않는 경우엔 64bit 광아우토반 호스팅 서비스를 신청해 주세요.

제로보드 XE버전을 사용 안하면

MySQL DB

  • MySQL DB용량은 하드(HDD)인 웹 용량과는 별개로 게시판 등을 이용하실 때
    텍스트의 총 용량만을 뜻합니다. 게시판에 속한 이미지나 동영상 등은 웹 용량에 포함됩니다.
  • MySQL DB용량은 서버 디스크 한도 내에서 무제한으로 제공되며 과도한 이용 시 제한될 수 있습니다.

트래픽(데이터 전송량) 용량 안내

트래픽

  • 트래픽이란 방문자가 고객님의 홈페이지에 접속함으로써 방문자에게 전송되는 데이터(텍스트, 이미지, 동영상 등)의 양을 뜻합니다.

트래픽 초과

  • 전송된 데이터 양이 호스팅 계정의 약정 트래픽 용량을 초과하게 되면 계정은 접속이 차단되며, "일일 트래픽 용량을 초과 하였습니다." 라는 문구가 포함된 안내 페이지가 출력하게 됩니다.
  • 웹호스팅의 트래픽 용량은 하루 기준이며, 매일 자정(00시)에 초기화되므로 트래픽 초과로 차단된 경우에는 자정이 지나면 자동으로 정상화가 됩니다.

트래픽 추가

  • 트래픽은 호스팅 서비스를 상위 사양으로 변경 및 옵션으로 추가가 가능하지만 무료 스트리밍과 무료 이미지CDN 서비스는 옵션 추가를 지원하지 않습니다.
  • 웹호스팅 서비스를 상위 사양으로 변경하시면 해당 사양에서 서비스되는 용량으로
    무료 스트리밍과 CDN 의 트래픽, HDD 용량은 함께 늘어납니다.

트래픽 용량 측정

  • 트래픽 용량은 일반적인 웹문서의 페이지당 권장량인 50K를 기준으로 20명이 접속했을 때 1000K=1M가 되므로 절약형 호스팅(트래픽 400MB)은 대략 8,000명이 50K 의 페이지를 볼 수 있는 용량이 됩니다.
  • 실제로 1페이지로 구성된 홈페이지는 거의 없고 페이지마다 용량이 모두 다르므로,
    몇명이 접속해야 트래픽 용량이 초과될지 정확한 수치를 안내해 드리기에는 무리가 있습니다.
  • 지속적으로 트래픽이 부족하다면 무료 서비스를 이용해 동영상이나 이미지 등을 분산하시거나, 한 단계씩 상위 사양으로 변경하시는 것도 하나의 방법입니다.

스트리밍 용량

  • 동영상/음악 파일(wmv,wma, asf, mp3)을 스트리밍으로 사용할 수 있는 무료 서비스 계정입니다.
  • 인기있는 동영상이나 음악 파일로 인해 일일트래픽 초과로 사이트 접속이 원할하지 않는 경우, 무료 스트리밍 호스팅에 파일을 올려 이용하시면 트래픽이 분산되므로 사이트 차단을 미연에 방지할 수 있습니다.

CDN 용량

  • 이미지 파일(JPG,GIF,PNG,BMP) 전용으로 이용하실 수 있는 무료 서비스 계정의 용량입니다.
  • 무료 스트리밍 서비스와 같이 이미지 사용량이 많은 경우, 웹 호스팅 계정과 분산하여 이용해 보시기 바랍니다.

DB

  • MySQL 4.X (EUC-KR 서버), MySQL 5.X (UTF-8 서버)을 기본으로 제공합니다.
  • MySQL DB용량은 하드(HDD)인 웹 용량과는 별개로 게시판 등을 이용하실 때
    텍스트의 총 용량만을 뜻합니다. 게시판에 속한 이미지나 동영상 등은 웹 용량에 포함됩니다.
  • MySQL DB용량은 서버 디스크 한도 내에서 무제한으로 제공되며 과도한 이용 시 제한될 수 있습니다.
  • PHP 7.x는 mariadb-10.0.x 를 지원합니다.

POP E-mail 계정 안내

  • 메일 계정은 아웃룩 등의 프로그램으로 설정하여 이용하실 수 있는 서비스로 계정당 5MB의 용량이 지원됩니다.
  • 메일 계정의 개수는 옵션으로 추가하실 수 있습니다.

도메인 추가 연결

  • 도메인 추가연결은 무료 도메인(ID.cafe24.com)을 제외한 개수이며, 고객님께서 보유중이신 도메인을 연결하실 수 있습니다.
  • 호스팅 계정에 연결된 도메인은 기본적으로 www폴더의 index 파일을 불러오게 됩니다.

설치비 안내

  • 고객님께서 신청하신 호스팅 서비스를 이용하실 수 있도록 서버에 계정을 설치하는것에 대한 비용입니다.
  • 비용은 서비스을 신청하실 때 한 번만 청구되며, 이후 연장에서는 추가로 납부하지 않으셔도 됩니다.
  • 사용중인 서비스를 상위 사양으로 변경하실 경우에는 변경되는 사양의 설치비 차액만 납부해 주시면 됩니다.
  • 설치비가 동일한 상품간(광아우토반 일반형 ~ 자이언트형)의 변경은 설치비가 동일하므로 청구되지 않습니다.

제로보드 설치 지원 안내

설치지원

  • 제로보드를 웹상에서 편리하게 설치하실 수 있도록 기능을 제공해 드린다는 의미이며, 카페24호스팅에서 제로보드를 설치해 드리는 것은 아닙니다.

버전

  • 일반광호스팅은 제로보드 4.X 까지 설치가 가능하며, XE 버전은 64bit 광아우토반 호스팅에서 설치가 가능합니다.

설치 안내

  • 제로보드 게시판의 개수 제한은 없으며, 호스팅의 용량 한도 내에서 게시판 생성이 가능합니다.
  • HP 게시판을 통해 자료를 업로드 하실 수 있는 용량은 10MB로 제한되므로 제로보드 게시판에 첨부하실 수 있는 용량도 10MB가 됩니다.

FTP/SSH 접속 안내

접속 정보

  • 접속은 계정 설치가 완료되면 바로 이용하실 수 있으며, 접속 정보는 고객님의 계정 아이디와 비밀번호, 연결된 도메인이 됩니다.
  • FTP/SSH의 계정 아이디는 변경이 불가능하며, 비밀번호는 [나의서비스관리>기본정보관리]에서 변경이 가능합니다.

접속 프로그램

  • 접속 프로그램은 상용화되어있는 FTP프로그램이나, 카페24호스팅에서 제공해 드리는 [부가서비스 > 웹FTP]를 이용하시면 됩니다
  • SSH 접속은 보안 기능이 강화된 SSH2만 지원하므로 PuTTY와 같은 SSH2를 지원하는 프로그램을 사용하시기 바랍니다. PuTTY 다운로드
  • FTP는 방화벽이나, 공유기 통해 인터넷이 연결된 PC에서 접속하신다면 접속하시는 해당 프로그램에서 Passive Mode를 체크 후 접속하십시오.

무료 스트리밍/무료 이미지CDN 안내

스트리밍이란?

  • 웹 상에서 동영상이나 음악 등을 실시간으로 재생하는 기법으로 대용량의 멀티미디어 파일을 제공할 때 유용한 서비스 입니다.

카페24의 무료 스트리밍

  • 카페24호스팅을 신청하신 분들께 무료로 제공해드리는 스트리밍 서비스 입니다.
  • 트래픽이 많이 발생하는 동영상, 음악 파일들을 무료 스트리밍 계정에서 제공함으로써 웹 페이지에 집중되는 트래픽을 분산시킬 수 있습니다.
  • 호스팅 신청 후 [나의서비스관리]에서 신청하실 수 있습니다.

카페24의 무료 스트리밍의 장점

  • 인기있는 음악 또는 동영상 파일을 홈페이지에서 제공할 경우, 증가하는 트래픽으로 인해 홈페이지 접속장애 또는 사이트 차단(일일 트래픽 초과)이 됩니다.
  • 음악 또는 동영상 파일을 무료 스트리밍 서버에서 서비스를 제공하여, 트래픽 증가에도 홈페이지 차단없이 안정적인 운영이 가능합니다.

CDN이란?

  • CDN(Content Delivery Network)이란 접속자가 인터넷상에서 가장 가까운 곳의 서버로부터 컨텐츠를 전송받도록하여 트래픽이 특정 서버에 집중되지 않고, 각 서버로 분산되도록 하는 기술입니다.

카페24의 무료이미지CDN

  • 카페24호스팅을 신청하신 분들께 무료로 제공해 드리는 이미지CDN 서비스입니다.
  • 트래픽이 많이 발생하는 이미지을 무료 이미지CDN 서버에서 서비스를 제공함으로써 웹 페이지에 집중되는 트래픽을 분산시킬 수 있습니다.
  • 호스팅 신청 후 [나의서비스관리]에서 신청하실 수 있습니다.

카페24의 무료이미지CDN의 장점

  • 접속자에게 가장 가까운 서버에서 컨텐츠를 전송 받기 때문에 빠른 서비스가 가능합니다.
  • 웹 페이지와 이미지 파일이 분산되어, 빠르고 안정적인 서비스가 가능합니다.
  • 다중 서버에 동일한 이미지 파일이 저장되므로 한 서버에 장애가 생기더라도 다른 서버에서 전송 받을 수 있으므로 안정적인 서비스가 가능합니다.
  • 접속자가 많은 대형쇼핑몰, 포털, 게임사이트, 이미지 서비스 사이트 등 이미지 파일을 많이 사용하는 고객님께 유용한 서비스입니다.
  • 지원되는 이미지 포맷은 JPG, GIF, BMP, PNG 이며, 지원 되는 포맷 외에는 서비스가 지원되지 않습니다.
    예를 들어 swf, wmv, mp3, zip 등의 파일은 사용하실 수 없습니다.
  • 개별 이미지의 파일 사이즈는 20MB를 초과할 수 없습니다.

  • 트래픽이 많이 발생하는 동영상, 음악 파일들을 무료 스트리밍 계정에서 제공함으로써 웹 페이지에 집중되는 트래픽을 분산시킬 수 있습니다.
  • 호스팅 신청 후 [나의서비스관리]에서 신청하실 수 있습니다.

카페24의 무료 스트리밍의 장점

  • 인기있는 음악 또는 동영상 파일을 홈페이지에서 제공할 경우, 증가하는 트래픽으로 인해 홈페이지 접속장애 또는 사이트 차단(일일 트래픽 초과)이 됩니다.
  • 음악 또는 동영상 파일을 무료 스트리밍 서버에서 서비스를 제공하여, 트래픽 증가에도 홈페이지 차단없이 안정적인 운영이 가능합니다.

하드(HDD) 용량

하드(HDD) 용량

  • 하드 용량이란 FTP를 이용해 윈도우호스팅 서비스에 웹 페이지 등의 파일을 올릴 수 있는 용량입니다.

용량 추가

  • 용량이 모두 사용된 경우에는 추가옵션 및 사양 변경으로 증설이 가능합니다.
  • 신청은 [나의서비스관리]에서 하실 수 있습니다.

트래픽(데이터 전송량) 용량 안내

트래픽

  • 트래픽이란 방문자가 고객님의 홈페이지에 접속함으로써 방문자에게 전송되는 데이터(텍스트, 이미지, 동영상 등)의 양을 뜻합니다.

트래픽 초과

  • 전송된 데이터 양이 호스팅 계정의 약정 트래픽 용량을 초과하게 되면 계정은 접속이 차단되며,
    "일일 트래픽 용량을 초과 하였습니다." 라는 문구가 포함된 안내 페이지가 출력하게 됩니다.
  • 웹호스팅의 트래픽 용량은 하루 기준이며, 매일 자정(00시)에 초기화되므로 트래픽 초과로 차단된 경우에는 자정이 지나면 자동으로 정상화가 됩니다.

트래픽 추가

  • 트래픽은 호스팅 서비스를 상위 사양으로 변경 및 옵션으로 추가하실 수 있으며, [나의서비스관리] 에서 신청이 가능합니다.

FTP 계정 지원

접속 정보

  • FTP는 윈도우호스팅 계정 설치가 완료되면 바로 접속이 가능하며, 접속 정보는 고객님의 계정 아이디, 비밀번호, 연결된 도메인이 접속 주소가 됩니다.
  • FTP의 계정 아이디는 변경이 불가능하며, 비밀번호는 [나의서비스관리 > 서비스접속관리] 에서 변경이 가능합니다.
  • 포트번호는 별도로 부여되므로 설치 완료 시 발송되는 메일이나, [나의서비스관리] 에서 확인하세요.

접속 프로그램

  • FTP 프로그램은 상용화되어있는 프로그램이나, 카페24호스팅에서 제공되는 [부가서비스 > 웹FTP] 를 이용하시면 됩니다.
  • 인터넷 공유기로 연결된 PC에서 접속하신다면 해당 FTP 프로그램에서 Passive Mode 를 체크 후 이용하십시오.

SFTP/FTP, SSH/Telnet 지원 안내

안전한 호스팅 환경 제공

  • 카페24호스팅은 보다 안전한 호스팅 환경을 제공하기 위해 기본적으로 SFTP 및 SSH를 지원합니다.

SFTP/SSH 안내

  • SFTP(Secure File transfer protocol)는 일반적인 FTP와 달리 PC와 서버간의 데이터 전송을 암호화하기 때문에 안전한 호스팅을 이용이 가능합니다.
  • 쉘 접속 또한 Telnet의 취약한 보안을 보완하기 위해 개발된 SSH(Secure Shell)를 지원합니다.

SFTP/SSH 접속

  • SFTP 접속은 FileZilla 등의 FTP프로그램이나, 카페24호스팅 [부가서비스 > 웹FTP]에서 이용할 수 있습니다.
    FileZilla 다운로드
  • SSH 접속은 보안 기능이 강화된 SSH2만 지원하므로 PuTTY와 같은 SSH2를 지원하는 프로그램을 사용하세요.
    PuTTY 다운로드

프로그램 자동설치 안내

프로그램 자동설치

  • 워드프레스, XE, 그누보드, 텍스트큐브, 킴스큐 등 설치형 프로그램을 자동으로 설치해드립니다.
  • 텍스트큐브 및 킴스큐는 PHP5.x 이하의 서버환경에서만 자동설치가 지원됩니다.

CharacterSet

  • 서비스 신청시 문자셋을 UTF-8로 선택해주세요.
  • 설치형프로그램은 UTF-8환경에서 자동설치가 가능합니다.

설치 안내

  • 신청한 호스팅의 세팅이 완료되면 [나의서비스관리 > 계정관리 > 프로그램 자동설치]에서
    한번의 클릭으로 자동 설치됩니다.
  • 제로보드 게시판의 개수 제한은 없으며, 호스팅의 용량 한도 내에서 게시판 생성이 가능합니다.
  • PHP 게시판을 통해 자료를 업로드 하실 수 있는 용량은 10MB로 제한되므로
    제로보드 게시판에 첨부하실 수 있는 용량도 10MB가 됩니다.

워드프레스 테마 제공

  • 워드프레스의 경우 [나의서비스관리 > 계정관리 > 워드프레스 테마 설치] 메뉴에서 다양한 테마를 설치할 수 있습니다.
  • 테마 설치 시 워드프레스 프로그램은 자동 설치됩니다.
  • 다양한 테마로 고객님의 원활한 비즈니스를 위한 홈페이지를 만들어 보세요.

Charset (캐릭터셋) 안내

Charset (character set, charset, 문자 집합)

  • 문자 집합은 정보를 표현하기 위한 글자나 기호들의 집합을 정의한 것입니다.
  • 문자나 기호의 집합을 컴퓨터에서 저장하거나 통신에 사용할 목적으로 부호화 하는 것을 문자 인코딩(부호화)이라 하고 인코딩 된 문자 부호(Character code)를 다시 디코딩(복호화)하여 본래 문자나 기호로 표현할 수 있습니다.

유니코드(Unicode) 프로젝트

  • 유니코드는 전세계의 모든 문자를 동일하게 표현하기 위한 산업표준으로 유니코드협회(Unicode Consortium)가 제정합니다.
  • 유니코드에는 ISO-10646에 포함된 문자집합, 문자 부호화와 문자를 표시하기 위한 복호화 알고리즘이 포함되어 있습니다.
  • ISO-10646은 문자 표시에 관한 국제 표준으로 초기 ISO-10646과 유니코드는 서로 다른 독자적인 표준이었으나, ISO-10646-1이 제정되면서 ISO실행위원회와 유니코드컨소시엄의 협의로 문자 표시 방법이 통합되어 현재의 국제 표준은 유니코드라고 할 수 있습니다.
  • ISO-10646에는 KS X 1001(KSC5601), EUC-KR, ISO-2022-KR의 한글 문자 부호가 포함되어 있습니다.

EUC

  • euc는 extend unix code의 약자로 유닉스에서 영어를 제외한 문자를 표시하기 위한 확장 부호를 의미합니다.
  • euc-kr은 한글 표현을 위한 문자 인코딩인데, 영문은 KSC5636(ASCII와 동일하나 역슬래쉬를 원표시로 대체)으로 처리하고 한글은 KSC5601로 처리합니다.
  • 과거 euc-kr은 KSC5601-87의 완성형 한글이었으나 현재의 euc-kr은 KSC5601-92로 조합형 한글까지 사용 가능합니다.

UTF-8 이란?

  • Unicode Transformation Format-8의 줄인말로 유니코드를 표현하는 방식의 하나로 전 세계의 문자를 공통적인 문자코드로 표현/관리하기 위한 규약입니다.
  • UTF-8의 가장 큰 장점은 모든 언어의 표현이 가능하다는 것이며, 다국어 홈페이지를 운영하실 고객님께서는 UTF-8을 이용해 사이트를 제작하시는 것이 좋습니다.
  • 현재 블로그 툴로 많이 알려져있는 테터툴즈, 워드프레스, 그누보드4 등의 프로그램은 UTF-8을 지원하는 서버에서 완벽하게 모든 기능을 사용하실 수 있습니다.

UTF-8 사용방법

  • UTF-8이 지원되는 64bit광아우토반FullSSD를 신청하신 경우, 서버의 기본 사양이 모두 UTF-8로 설정되어 있습니다.
  • 정상적으로 사용하시기 위해서는 작업하신 소스를 저장할 때 반드시 UTF-8형식으로 저장하셔야 합니다.
  • UTF-8형식으로 저장하지 않을 경우, 한글 및 외국어 표기가 정상적으로 출력되지 않을 수 있습니다.
  • html 작성 시에는 meta tag의 charset을 UTF-8로 지정하십시오.
예)
<head> <meta http-equiv="Content-Type" content="text/html; charset=utf-8" /> </head>

네트워크 용량

네트워크 용량이란?

  • 방문자가 고객님의 홈페이지에 접속함으로써 방문자에게 전송되는 데이터의 양입니다.
  • 방문자가 사이트 방문/이미지/음악/동영상 파일등을 재생 및 다운로드 할 때 발생하게 됩니다.
  • 데이터 사용량이 호스팅의 약정 네트워크 용량을 초과하면, 초과된 용량만큼 별도로 결제하셔야 합니다.
  • 초과된 용량에 대해 미결제 시 서비스 차단이 발생할 수 있습니다.
  • 카페24에서는 고객님의 서버가 안정적으로 유지될 수 있도록 충분한 여유 대역폭을 확보해 두므로 대역 제한을 하지 않습니다.

초과 네트워크 요금 납부

  • 기본 대역 10M ~ 30M를 초과한 네트워크 요금은 다음달에 후불로 납부하시면 됩니다.

네트워크 용량 측정

  • 네트워크 용량은 웹호스팅의 트래픽과는 달리 누적으로 계산되지 않으며, 동시에 접속할 수 있는 양으로 측정됩니다.
  • 예시) 단독웹호스팅 베이직 서비스
    • 이미지 파일 500K를 기준으로 20명이 접속했을 때 10,000K (10M)가 됩니다.
    • 20명이 접속해있는 순간 약정 네트워크 10M가 발생하는 것입니다.
    • 21명째 방문자가 접속을 시도할 경우 추가 트래픽이 발생하며, 추가분에 대해서 별도의 요금이 청구됩니다.

초과 네트워크 요금 계산 방법

  • 계산 기준
    • ⓐ 사용기간(과금대상기간)의 5분단위 트래픽 측정값 중 최대값
    • ⓑ 사용기간(과금대상기간)의 총 평균 트래픽의 3배
  • 계산 방법
    • ⓐ, ⓑ 중 최소 트래픽 값을 계산 합니다.
    • 초과금액 = (총사용트래픽 - 기본트래픽) * 17,000원(1Mbps당) (VAT 별도)

DB Server 추가에 대한 안내

DB Server 추가 및 서버 이중화

  • 대량 DB처리 및 접속자 증가로 서버 리소스가 부족하게 되면 사이트가 느려질 수 있습니다

SFTP/SSH 안내

  • 쉘 접속 또한 Telnet의 취약한 보안을 보완하기 위해 개발된 SSH(Secure Shell)를 지원합니다.

SFTP/SSH 접속

  • SFTP 접속은 FileZilla 등의 FTP프로그램이나, 카페24호스팅 [부가서비스 > 웹FTP]에서 이용할 수 있습니다.
    FileZilla 다운로드
  • SSH 접속은 보안 기능이 강화된 SSH2만 지원하므로 PuTTY와 같은 SSH2를 지원하는 프로그램을 사용하세요.
    PuTTY 다운로드
  • 일반 FTP와 Telnet 접속도 가능하지만 안전한 호스팅 서비스를 원하신다면 SFTP와 SSH 사용을 권장합니다.

트래픽 용량(데이터 전송량) 이란?

트래픽

  • 트래픽이란 방문자가 고객님의 홈페이지에 접속함으로써 방문자에게 전송되는 데이터(텍스트, 이미지, 동영상 등)의 양을 뜻합니다.

트래픽 초과

  • 전송된 데이터 양이 호스팅 계정의 약정 트래픽 용량을 초과하게 되면 계정은 접속이 차단되며, “일일 트래픽 용량을 초과 하였습니다.” 라는 문구가 포함된 안내 페이지가 출력하게 됩니다.
  • 웹호스팅의 트래픽 용량은 하루 기준이며, 매일 자정(00시)에 초기화되므로 트래픽 초과로 차단된 경우에는 자정이 지나면 자동으로 정상화가 됩니다.

트래픽 추가

  • 트래픽은 호스팅 서비스를 상위 사양으로 변경 및 옵션으로 추가가 가능하지만 무료 스트리밍과 무료 이미지CDN 서비스는 옵션 추가를 지원하지 않습니다.
  • 웹호스팅 서비스를 상위 사양으로 변경하시면 해당 사양에서 서비스되는?용량으로 무료 스트리밍과 CDN 의 트래픽, HDD 용량은 함께 늘어납니다.

트래픽 용량 측정

  • 트래픽 용량은 일반적인 웹문서의 페이지당 권장량인 50K를 기준으로 20명이 접속했을 때 1000K=1M가 되므로 절약형 호스팅(트래픽 400MB)은 대략 8,000명이 50K 의 페이지를 볼 수 있는 용량이 됩니다.
  • 실제로 1페이지로 구성된 홈페이지는 거의 없고 페이지마다 용량이 모두 다르므로, 몇명이 접속해야 트래픽 용량이 초과될지 정확한 수치를 안내해 드리기에는 무리가 있습니다.
  • 지속적으로 트래픽이 부족하다면 무료 서비스를 이용해 동영상이나 이미지 등을 분산하시거나, 한 단계씩 상위 사양으로 변경하시는 것도 하나의 방법입니다.

스트리밍 용량

  • 동영상/음악 파일(wmv,wma, asf, mp3)을 스트리밍으로 사용할 수 있는 무료 서비스 계정입니다.
  • 인기있는 동영상이나 음악 파일로 인해 일일트래픽 초과로 사이트 접속이 원할하지 않는 경우, 무료 스트리밍 호스팅에 파일을 올려 이용하시면 트래픽이 분산되므로 사이트 차단을 미연에 방지할 수 있습니다.

설치비 안내

Inno DB 안내

  • InnoDB는 MySQL의 트랜잭션 세이프(transaction- safe) 스토리지 엔진으로 대용량 데이터를 처리할 때 최대의 성능을 내도록 설계되었습니다.
    트랜잭션 처리를 통해 다중 쿼리 실행 중 실패하는 경우 롤백(Rollback)이 가능합니다.
  • MySQL DB용량은 서버 디스크 한도 내에서 무제한으로 제공되며 과도한 이용 시 제한될 수 있습니다.
