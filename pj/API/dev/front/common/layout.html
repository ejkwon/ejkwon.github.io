API에 대한 호출방식은 HTTP 호출(Request)/응답(Response) 규약에 의해 호출되고 결과를 회신받을 수 있습니다

HTTP 호출(Request) / 응답(Response) 개요

카페24 API의 호출과 응답은 HTTP Request / Response 기본 규약에 따라 정의되어 있습니다.

1. HTTP Request

  • 호출(Request)란, API를 통해 쇼핑몰의 데이터를 제어/조작(조회, 입력, 수정, 삭제)하기 위한 요청을 하는 것을 말합니다.
  • 호출(Request)를 위해서는 도메인 + 파라미터 형식을 이용하는데 파라미터는 기본적으로 key = value 방식입니다. 값이 여러개인 경우 콤마로 구분합니다.
  • 기본적으로 API에서 제공되는 기본 규약은 아래와 같습니다.
상세설명
구분 설명 상세설명
GET 조회/상세 데이터를 조회(상세조회)하는 역할을 수행합니다.
POST 조회/상세 여러줄 여러줄 여러줄여러줄 데이터를 수정하되 데이터가 존재하지 않는 경우 등록하는 역할을 수행합니다. (예제 : 상품이미지 수정 등)

2. HTTP Response

  • 응답(Response)란, API에 호출(Request)된 내용에 대한 처리결과를 반환하는 것을 말합니다.
  • 응답(Response) 결과는 key : value에 대한 리소스 묶음으로 구분되어 반환됩니다. 리소스명은 Instance / Collection에 따라 단수/복수명사가 변경됩니다.

OAuth 개요

  • OAuth 2.0은 사용자 인증을 위한 글로벌 산업 표준 프로토콜입니다.
  • 특정 어플리케이션에서 쇼핑몰에 로그인하기 위해서는 사용자 ID, 비밀번호가 필요합니다. 하지만 매번 쇼핑몰 운영자가 ID, 비밀번호를 알려주기가 어렵습니다.
  • 이때 필요한 인증 방식이 OAuth를 통한 인증과 권한부여 입니다. OAuth 인증을 사용하는 경우 어플리케이션은 운영자 ID에 해당하는 키(Key)와 비밀번호에 해당하는 시크릿 키(Secret Key)를 통해 쇼핑몰 서버로 부터 인증을 받아 쇼핑몰의 정보를 입력, 수정, 조회, 삭제 할 수 있게됩니다.
  • OAuth는 웹, 데스크탑, 모바일 등의 어플리케이션에서의 인증(Authorization)을 정의하고 접근(Access)권한을 관리합니다.
  • 보다 자세한 OAuth에 인증에 대한 내용은 oauth.net을 통해 확인해 볼 수 있습니다.
