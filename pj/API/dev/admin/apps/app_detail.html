  • App의 기본정보(이름, 서비스URL)과 API를 사용하기 위해 필요한 정보(Redirect URL(s), 권한(Scope) 등)을 입력하거나 수정할 수 있습니다.
  • App의 인증정보 (API key, API secret key, API token 정보는 App의 인증 및 동작에 중요한 정보이오니 관리에 주의해야 합니다.

기본정보

기본정보
App 이름
  • App 이름은 App이 판매되기 전까지 수정이 가능합니다.
App URL
  • App을 서비스하기 위한 URL을 입력해 주세요. (프로토콜, 포트의 지정이 필요한 경우 함께 입력해야 합니다.)
Redirect URL(s)
Redirect URL(s)
  • App이 인증 후 redirect_uri 로 전달될 URL을 입력할 수 있습니다.
  • URL의 구분은 엔터(enter)로 구분하여 입력합니다. (최대 10개 / 200byte 이내)
  • 보안정책 기준으로 https를 반드시 사용해야 합니다. (http는 지원되지 않습니다.)
  • 서브 도메인이 있으면 대표 도메인명만 입력해 주세요. (예: http://cafe24.com)
App 대표아이콘
  • App 대표아이콘은 사용자들이 App을 구분할 수 있고 홍보하는데 도움이 됩니다.
  • 권장(등록) 이미지: 512px * 512px / 500KB이하 / png, jpg(jpeg)

선택된 권한

선택된 권한
No 분류 권한 적용범위 설정
1 앱 읽기 + 쓰기권한 앱 설치와 관련된 정보를 조회할 수 있습니다. -
2 상품 상품(Products) 정보
  • App이 동작하는데 필요한 API 권한을 체크하여 획득할 수 있습니다. (권한사용에 대한 법적 책임은 App 개발사(자)와 구매자에게 있습니다.)
  • App의 동작과 무관한 권한을 획득하는 경우 App의 심사나 판매를 거부할 수 있으니 꼭 필요한 권한만 선택하여야 합니다.
  • 선택된 권한은 App 구매시 구매자의 동의를 받게되며, 구매자가 권한사용을 동의하지 않는 경우 App은 설치/판매되지 않습니다.
운영자 권한확인 URI
  • 쇼핑몰의 운영자/부운영자가 제어할 수 있는 App의 화면 또는 기능의 권한을 확인할 수 있는 URL을 입력할 수 있습니다.
  • URI을 통해 전달된 App의 권한 내역은 쇼핑몰 솔루션에서 운영자 설정 권한으로 제공되며, App에서 운영자별 권한을 제어할 수 있도록 제공됩니다.
  • 자세한 권한 설정 URL을 생성하기 위한 방법은 'App 개발가이드'를 참고하여 주시기 바랍니다.
