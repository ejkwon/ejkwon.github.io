컨텐츠의 구성

컨텐츠의 구성
명칭 정의 규칙
page 사용자가 웹사이트에서 보이는 한 화면단위이며, 즉 하나의 html 이며,
제목+ 컨텐츠 + 전체를 컨트롤 하는 버튼으로 구성 되어있다. 		-
section 모듈의 모음으로 한 row 단위로 구성된다.
해당 class는 유일하게 사용하며, 다른 어떤 요소에도 사용을 금한다. 		.section
module 기능의 단위를 의미하며, 프로그래밍적인 의미의 기능과 요소의 묶음을 포함한다. (ex, 타이틀) 모듈의 정의(우선순위)
  1. 1. 공통으로 사용 될 경우 모듈로 처리한다.
  2. 2. 기능 혹은 데이터로 처리할 경우 모듈로 처리한다.
  3. 3. 그 외 단순 컨텐츠 일 경우 : 모듈에서 제외
 일반모듈: .m + [명시적인단어] (예: mTab)
알럿창용 모듈: mAlert + [명시적인단어] (예: mAlertTab)
레이어팝업용 모듈: mLayer + [명시적인단어] (예: mLayerTab)

화면 구성

컨텐츠의 배치, 여백을 정의하며 쓰임새에 따라 아래 4가지 형태로 분리된다.

화면 구성
명칭 정의 규칙
area 배경이나 float해제 relative min-height 등 디자인을 위한 빈 div가 필요한 경우가 발생시, area로 그룹핑 하여 사용한다. 모듈을 감싸는 경우: [명시적인단어]+Area
개발이 필요한 기능의 단위가 아닌, 컨텐츠 단위의 묶음이 필요할때 사용한다. [명시적인단어]+Area
모듈안에 사용 될때는 .inner로 통일한다. 모듈 내부에 위치하는 경우: .inner
column 한 Row에 하나의 섹션(모듈)이 삽입되지 않고, 2개 이상의 섹션(모듈)이 배치될때 사용된다. 섹션(모듈)에만 적용 할 수 있으며, 단순 엘리먼트 단위에선 사용 될 수 없다. .column1 ~ .column16
grid 모듈단위가 아닌 엘리먼트 단위의 요소의 묶음을 포함하는 경우에 사용된다.
부모요소의 float clear를 위해 사용되거나 배경등이 필요할때 사용된다. 		.g[명시적인단어]의 조합
모듈이 아닌 엘리먼트의 여백을 구하고자 할때 사용된다. 모듈이 여백을 가질때는 모듈의 class에서 여백을 지정한다.
여백은 bottom 에 정의하는 것을 우선으로 하며, 특수한 경우에만 방향을 지정하는 class를 추가한다. 		.gSingle, gDouble, gTriple, gSingleTop
툴팁이나 레이어팝업등 동일한 UI에서 컨텐츠 폭을 Variation하기 위해 사용된다. .gSmall, .gMedium, .gLarge

이벤트 적용
명칭 정의 규칙

javascript의 trigger로 사용되며, 어떠한 디자인 속성도 가지지 않는다. 		event의 e를 접두어로 사용한다.
